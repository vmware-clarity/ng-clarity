Alerts are banners that draw the user's attention to an important message. Elements of an alert, such as icons and color, indicate the type and urgency of the message's information.

Usage

Use alerts to grab the user’s attention to provide critical information needed in context.

In Cards

You can use an alert at the very top of a card, above the card title.
See the following best practices for using alerts in cards:

  • Use concise language within the alert.
  • Use a compact-size alert. The focus of the alert should be on its content, not on the alert appearing in it.
  • Place the alert at the very top of a card, before the title. An alert is meant to attract the attention of the user.
  • Don’t use more than one alert in cards, if possible. Seeing more than one alert within a card distracts the user and dilutes the importance of the alerts displayed.

App-Level Alert

Use app-level alerts in the global context of an application. Place them above all content and navigation.

Status

There are three different statuses of app-level alerts: Error, Warning, and Info. App-level alerts follow the same urgency order as standard alerts: error, warning, and info.

Why no success app-level alert?

App-level alerts are global in nature. To report the success of an operation, communicate that within context or as a notification message.

Use for malfunctions and critical issue, such as license expiration.

Size

Use the .alert-sm class with .alert for an alert 24 pixels in height. Or use .alert-lightweight for a borderless lightweight alert. The default is 36 pixels.

App-Level Alerts

Use the .alert-app-level class with .alert to render an app-level alert.

CSS

Several classes and elements are required to implement the Clarity alert layout. A table of these classes and elements with a brief description of each follows:

.alert This class is a wrapper around .alert-items and the .close button. Place the .close button before the alert items.
.alert-items This class is a wrapper around one or more .alert-item elements.
.alert-item This class is a wrapper around .alert-text, .alert-icon-wrapper, and .alert-actions.
.alert-icon-wrapper .alert-icon-wrapper contains a cds-icon with the classname .alert-icon applied to it. The icons used for the different alert types of success, danger, warning, and info are, respectively: success-standard, error-standard, warning-standard, and info-standard.
.alert-actions .alert-actions can consist of dropdowns or links. Each action should extend the .alert-action class.
