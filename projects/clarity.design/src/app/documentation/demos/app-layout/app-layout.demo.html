A properly structured layout enforces an optimal, consistent experience across your application.

Layout

A Clarity application layout includes a main container and content container.

Main Container

The main container (.main-container) is a vertical flexbox that wraps the following components:

  • App-level alerts
  • Header and any sub navigation under the header
  • Content container

For information on navigation components, header, sub navigation, see Navigation.

Note: Although Clarity does not have a footer component, a custom footer can be added in the main container.

Content Container

The Content Container (.content-container) is a horizontal flexbox that wraps the following components:

  • Content Area
  • Vertical Nav

Basic Structure

Two constants of an application built in Clarity are the header and content area. These are the blocks upon which you build your application model.

Header

The header is for branding and app-level elements, such as navigation, search, and account settings. See Header for more information.

Content Area

The content area is where users focus their attention most of the time, gathering information and performing tasks–it is the canvas for your application. As the largest portion of your application, the content area is always visible.

Content Area Layout

Your layout should reflect the information or workflow of the selected navigation. When laying out the content, keep the following considerations in mind:

  • The flow of content, specifically how to create a hierarchy and layout that draws attention to the areas of importance
  • The importance of designing to the grid
  • How to help users complete their tasks
  • How to handle large amounts of data
  • Responsive design (if that is part of your product’s goals)

Content can consist of Clarity components, or no components and just information. The following sections cover common layout patterns and usage recommendations.

Cards

Cards present high-level information and guide users to related actions and details. Cards can include a combination of text, images, and data visualisations.

Benefits of using cards include:

  • Helps to see data in a collection
  • Facilitates the scanning of information
  • Works well across platforms

Tables and Datagrids

Tables and datagrids are good for managing large amounts of data. These layouts work well when users need to compare data and perform batch operations.

A table is a static view. A datagrid provides users flexibility to view the data, including filtering and sorting.
Complex tables and datagrids work best on larger screens.

Forms

Forms collect data from users. Forms are comprised of other components, including labels, input fields, labels, check boxes, radio buttons, and text.

Forms allow user to quickly see what information they must provide. However, too many fields can discourage the user.
Inline forms are better than modals when you don't want to block users from performing other actions.

Tabs

Tabs appear in a single, non-scrollable row at the top of the content area. They are good for breaking content into separate, related views.

Tabs are not appropriate if users need to compare data across views.

White Space and Typography

White space and typography are important elements in conveying hierarchy. These elements direct users to what they should view next and make the content and data easier to parse. They also help bring consistency to an app.

Button Placement

In the content area, buttons are left-aligned, with the primary button in the leftmost position. This placement supports the F-pattern layout.

Using Vertical Rhythm for Layout

Vertical rhythm is the repetition of spatial relationships in a design. A consistent rhythm gives elements a uniform and balanced placement in a design. The more consistent the design, the easier it is for users to read and understand.

The Clarity Baseline is 24px
All elements in Clarity are designed with a 24px baseline:

  • The visual heights of all components, line heights and spacing is divisible by 4.
  • The vertical white space between elements is in multiples of 24px.

Repeat 24px in Your Layout

Design the vertical margins and padding between elements using the Clarity baseline. A multiple of 24px can be a whole or fractional as long as it’s divisible by 4. For example: 4px, 8px, 12px, 16px, 20px, 24px, 28px, 32px, 36px, etc.
