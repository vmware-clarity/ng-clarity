The Clarity color system is based on a set of global colors that support Clarity’s default light and dark themes and charts. The colors can also be used as spot color in your application or to design your own custom theme. Clarity prefers HSL color definitions.

Usage

Clarity uses a functional naming system to tie colors to their purpose. For example, info blue is HSL: (198, 100%, 34%) which is also referred to as --cds-global-color-blue-700 and, when used as the default info color, by its alias --cds-alias-status-info. The system extends the status colors since these components often require more than one color to enable interactive changes and multicolor designs. A similar system is used for interaction styles in all non-status components. Typography also follows these same conventions.

By using this system throughout your application, especially in cases where you design custom components, you can ensure consistency with Clarity components while remaining accessible and theme-able. Using this system also ensures future changes in Clarity will be automatically applied to your application. This greatly reduces the effort required to stay current with the latest version of Clarity.

We provide a detailed specification both here and in storybook, and suggest becoming familiar with the way it works to avail yourself of the benefits.
