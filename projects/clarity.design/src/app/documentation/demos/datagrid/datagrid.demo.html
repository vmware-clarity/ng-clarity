Datagrids organize large volumes of data in columns and rows allowing users to scan, compare, and perform other actions.

Usage

The Datagrid component works best for structured and large volumes of data.

For Structured Content

Datagrids work best for structured, homogeneous content, where each object has the same attributes. When common attributes are directly aligned in columns, users can quickly scan and compare them.


For data sets with a blend of text, images, and data visualizations, or content with mixed formatting, cards offer a better layout.

For Large Volumes of Data

A datagrid is well-suited for presenting large volumes of data that doesn’t fit on one page. Users can filter and sort the data according to preference.


For smaller amounts of data (10 to 20 lines), use a table. However, you still consider using a datagrid under the following circumstances:

  • The data set will grow.
  • Users need search, filter, or batch operations.

You can allow Clarity to handle all the data processing needed by the various features or you can customize presentation and behavior.

Pagination

Since the point of a datagrid is to display large amounts of data, often you cannot display it all on the same page. To solve this, our datagrid supports pagination.

Benefits

With pagination the user can control the number of rows per page and can carefully review the information page by page.

Virtual Scrolling

With virtual scrolling users can vertically scroll through all records in the datagrid instead of navigating through pagination. Unlike infinite scrolling, which continuously adds new rows, virtual scrolling loads new rows and removes previous rows from view, improving rendering performance.

Loading Methods

  • Server-side loading: Fetch content from the server in chunks during scrolling.
  • Client-side loading: Load all content at once and store it in the browser.

Usage

Use virtual scroll instead of pagination when you want to speed up users’ workflow.

Benefits

With virtual scroll the user can quickly glance through large amounts of information and that way we may speed up their workflow in in cases where we value more speed over details.

Considerations

  • Select all functionality is not supported with virtual scrolling.
  • Client-side loading may impact initial load time and browser performance.

Accessibility

  • Keyboard-only users: virtual scrolling may pose challenges.
  • Consider providing a downloadable file for better accessibility.

Detail Pane

Clarity offers a detail pane to show additional details for a record. The detail pane condenses the datagrid to show a primary column and a panel to the right, which has more details about the record. This allows you to show the full content of the record in a larger scrollable space. The detail pane is also fully accessible for keyboard and screen reader users.

Edge cases:

  • Hide the details pane icon for records without additional details when we have a mix of some records with and others without additional details.
  • Use a disabled icon when we have details, but the user has no permission to see them.

Expandable row

Use expandable rows when you have additional information for a row, or row cells that do not need to be shown at all times. This helps minimize visual clutter. It also frees up the need of a second datagrid that gets updated with details. This is sometimes called a master-detail or master-child pattern.


Another use is replacing original row data with a customized view or layout that includes most or all of the original row data. The expanded area can be loaded with other components as well to fit your needs.
