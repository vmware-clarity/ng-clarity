Date Range Picker is a popover control that combines two date inputs for selecting a range of date values.

Usage

Use the Date Range Picker component to allow selection of a range of date values. This helps the user see and select the date in context with other possible dates. Date choices are influenced by day of the week, or week within a month. A Date Range Picker is also helpful when the formatting of the date is important.

Consider omitting the Date Range Picker component in the following scenarios:

  • When the user does not require to select custom range from calendar and has static predefined date ranges. Then provide predefined date ranges using the select component.
  • When space is limited.

Behavior

The user can open the Date Range Picker popover by clicking, or gaining focus and hitting enter, on the calendar icon next to the input field.

Today’s date is displayed in bold.

Clicking on the check-calendar icon inside the Date Picker returns the focus to today’s date.

Focus is indicated by blue bordered line around the date. Focus can be moved around inside the Date Range Picker using arrow and tab keys. Clicking the enter key will select a value.

Views

The Month and Year views are shown when a user clicks the month or year titles in the date view.

Month

Monthly values can be selected when the user clicks the month title in the date view. Selecting a month value returns the user back to the date view.

Year

Ten years are displayed at one time. Arrow keys can be used to navigate through the values: future years and past years. Selecting a year value will return the user back to the date view.

Small Screens

Clarity Date Range Picker is only shown when screen width is 768px and above.

We automatically disable the Clarity Date Picker on small mobile phones. This is done because, unlike desktop browsers, most mobile browsers have built-in date pickers. These built-in date pickers are optimized to work on mobile devices and provide an experience that is familiar to the user.

Input Behavior

Every time the user changes the value in the input field, the value is validated against the placeholder to check if a valid date is entered. If a valid date is entered, the date range picker opens with the entered date selected. If an invalid date is entered, the date range picker opens with the focus on today's date.

Selecting a date from the date range picker overrides the value in the input field.

Date range picker ideally requires a 4 digit year but can convert a 2 digit year input by the user to a 4 digit year. For example, a 2 digit year 20 will convert to 2020. If the 4 digit conversion is 20 more than the current year, we subtract 100 from it. If the current year is 2018 and the year input is 99, then the converted year is 1999. Anything other than a 4 digit or 2 digit year is considered invalid.

Accessibility

For applications that use the ClrStartDateInput / ClrEndDateInput directive and the associated ClrDateRangeContainer component there is built in support that adds accessible behavior to the control and its form. This behavior includes:

  1. Programmatically associating the correct label for attribute with the id of the input
  2. Automatic wiring up of the aria-describedby behavior with associated clr-control-error elements
  3. Adding the label to a general form summary for screen readers when the control is in an error state after a form is submitted

Code & Examples

Angular Component

<clr-date-range-container> component

Input/Output Values Default Effect
[min] string N/A Sets the earliest acceptable date and must be in the YYYY-MM-DD format.
[max] string N/A Sets the latest acceptable date and must be in the YYYY-MM-DD format.
[rangeOptions] DateRangeOption[] [] Predefined Date range options.

[(clrStartDate)] directive

Input/Output Values Default Effect
[(clrStartDate)] Boolean false Two-way binding on the start date value.
(clrStartDateChange) Date N/A emits when the start date value is changed

[(clrEndDate)] directive

Input/Output Values Default Effect
[(clrEndDate)] Boolean false Two-way binding on the end date value.
(clrEndDateChange) Date N/A emits when the end date value is changed
