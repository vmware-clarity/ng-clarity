Density

Clarity introduces a density model that enables applications to adapt layout and component sizing based on user needs or platform constraints. By utilizing density tokens in combination with existing design and component tokens, applications can seamlessly toggle between regular and compact UI experiences.

Density Modes

The density model offers two primary modes:

  • Default – optimized for readability and comfort, with ample spacing ideal for touch devices and desktop applications.
  • Regular – Will replace "Default" in next major version although it's the same as default plus little optimisations in terms of row heights from 36px to 32px.
  • Compact – optimized for data-dense views, reducing spacing and component padding to maximize information visibility in constrained spaces.

These modes are applied globally and can be toggled dynamically, allowing responsive experiences across different form factors and use cases.

Using Density Tokens

Density tokens extend the existing space and sizing token system in Clarity. Instead of fixed values, density-aware tokens adjust component padding, spacing, and sizing based on the selected density mode. This allows consistent behavior across themes and layouts.

Common density token categories include:

  • --clr-base-typography-font-size-* – font-sizes
  • --clr-base-typography-line-height-* – line-heights
  • --clr-base-border-radius-* – border-radius
  • --clr-base-vertical-offset-* – vertical offsets
  • --clr-base-horizontal-offset-* – horizontal offsets
  • --clr-base-gap-* – layout gaps
  • --clr-base-icon-size-* – icon sizes
  • --clr-base-layout-space-* – general-purpose spacing

These tokens automatically resolve to their appropriate value depending on the active density context.

Design Considerations

Components and layouts built using space and density tokens will automatically respond to density changes. However, if you're implementing custom components, ensure they consume these tokens rather than hardcoded values.

The compact density mode is especially useful in table-based layouts, dashboards, or tools where screen real estate is limited. The regular mode is preferable for forms, content-heavy views, or accessibility-focused designs.

Accessibility and Density

While compact layouts increase information density, it is important to validate accessibility, especially for touch targets.

Always test all density modes with real users to ensure usability and accessibility are preserved.

Applying Density

Density is applied using the clr-density attribute. This attribute can be placed on the <body> element to define a global layout mode, or on individual containers to scope compact or regular density locally within your application.

The supported values are:

  • default – default layout with generous spacing for readability and comfort.
  • regular – similar to default layout but with lowered row height. 36px -> 32px
  • compact – reduced spacing and control sizes, suitable for data-dense interfaces.
