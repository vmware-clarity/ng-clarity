The header is the component at the top of every page of your application. Headers provide a consistent place for branding, navigation, search, and access to global application actions such as settings and notifications.

Style

The header is consistent across all pages of an application. This offers a recognizable unified experience for the application.

Branding

Include both your application’s name and logo if available. Clicking on either of these take you to the homepage of the application.

Top-Level Navigation

Include top-level navigation items in the header. These items should be persistent throughout the user’s experience.
Have a maximum of four items in top-level navigation. For more info, take a look at Navigation.

Typography

For top-level navigation items in text, use clear language. Although using icons instead of text is often not as helpful to users, if you do use icons, add tooltips to help users understand what the icons mean.

Do not combine icons and text unless the text provides some user context.

Responsive Navigation

Clarity includes functionality for displaying the navigation on smaller devices like tablets and phones with the responsive navigation component in clarity-angular. For more information, see Responsive navigation in the Navigation component documentation.

Search

There are many different ways to include search in the header. Which way you choose depends on search's prominence and its utility within the application.
You can read more about search in the design spec. The search component itself is a work in progress.

High profile

Use search as a prominent navigation item in the header if you are building a search-based user experience.

Low profile

Use search as a secondary item to the right of the header if search is not core to your navigation experience or if you need to emphasize three or more top-level navigation items.

Right-Side Action Area

The area to the right of the header serves as a location for global actions and top-level navigational items that are important, yet visited less frequently.

For example, the action area could include application settings, user settings, help, about, and other similar items.

CSS Header Component

CSS Classes

.header

.header is a wrapper around the following four sections:

  • Branding
  • Navigation
  • Search
  • Settings
.branding

.branding contains the product logo and the product title. The logo extends the .cds-icon class and the title extends the .title class.

.header-nav

.header-nav contains the navigation links. Each navigation link extends the .nav-link class along with the .nav-text class for text links and the .nav-icon class icon links.

.search

.search is a form containing the search icon and the search input field.

.header-actions

.header-actions is a wrapper that contains secondary navigation links. Each navigation link extends the .nav-link class. Navigation links can be text or icons.

For information about headers with responsive navigation, see Responsive Navigation.

Accessibility

To let the assistive technologies better track the state of the navigation we recommend adding aria-current="page" to the active link. You will need to add it or remove it depending on which link is currently active. A good marker for that will be the element that is having the class .active on it. The clrAriaCurrentLink directive can be added to any link that uses the routerLink directive to accomplish this.
