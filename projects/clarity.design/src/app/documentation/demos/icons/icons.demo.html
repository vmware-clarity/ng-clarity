Clarity icons provides pixel-perfect, scalable SVG-based icons. The icon system gives you complete control over icon color, orientation, and size. Additionally, you can access and customize any SVG graphic elements inside the icon through standard CSS.

Usage

Follow the instructions in Getting Started: Developing to install the @cds/core package.

Import

To start rendering SVG shapes or providing custom SVG shapes, import the icon with the icon service.

Instantiate

Add an icon element to the application templates as needed and specify a registered shape.
Tip: You can browse the collection of shapes included with Clarity.

Customize

You can add your own custom SVG shapes if the Clarity icon library doesn't have exactly what you need. You must provide a unique shape name and the SVG for its shape.

After the custom shape is added, use it just like any other icon.

Behavior

Icon size

By default, the size for Clarity icons is 16 by 16 pixels. You have two ways of customizing this default value:

  • Use the size attribute.
  • Manually customize the width and height values in your icon's style.

The size attribute takes a string number or one of the standard Clarity t-shirt sizes.

Rotation

The default rotation for every shape that comes in the Clarity icon library is 0 degrees. With directional icons such as arrows and carets, 0 degrees means that it points up. To rotate an icon to face another direction, specify this using the direction attribute. Alternatively, you can set the rotation values in the icon's style attribute.

Flips

You can flip icons horizontally or vertically through the use of the flip attribute.

Icon Colors

Clarity icons also come with predefined colors using the status attribute. Colors are described as:

  • Danger: Use this color when you want to show errors or make the user pause and evaluate.
  • Warning: Use this color when you want the user to proceed with caution.
  • Success: Use this color when you want the user to know that something is correct.
  • Info: Use this color when you want the user to be informed.

For best accessibility remember that the color alone is an error since it is not sufficient to communicate status. Using an icon shape that clarifies the status associated with the color fulfils this requirement.

There is also an inverse attribute, which is suitable for icons used on contrasting backgrounds.

Badges and Variants

Clarity icons come with variants for badges that add visual indicators to the badge, where applicable.

Badge shapes and colors

  • Info (circle): Use this variant when you want the icon to inform.
  • Success (circle): Use this variant when you want the icon to show something is correct.
  • Danger (circle): Use this variant when you want this icon to show something is incorrect.
  • Warning (circle): Use this variant when you want to show something needs caution.
  • Danger (triangle): Use this variant when you want this icon to show something is incorrect.

Note: Not all icons support all variants. Check the details for specific shapes to see the variants that it supports.

Icon Properties

Name Type Description
shape string Changes the SVG glyph displayed in the icon component.
Defaults to the 'unknown' icon if the specified icon cannot be found in the icon registry.
size string | 'xs' | 'sm' | 'md' | 'lg' | 'xl' | 'xxl' Applies numerical width-height or a t-shirt-sized CSS class name.
direction 'up' | 'down' | 'left' | 'right' Applies a directional value that rotates the icon 90° with the top of the icon pointing in the specified direction.
flip 'horizontal' | 'vertical' Applies an orientation value that flips the orientation of the icon vertically or horizontally.
solid boolean Displays most icons in their "filled" version if set to `true`.
status 'info' | 'success' | 'warning' | 'danger' | 'neutral' Changes the fill and outline color of the icon.
inverse boolean Inverts the fill and outline color of the icon if `true`. Useful to display icons on a dark background.
badge 'info' | 'success' | 'warning' | 'danger' | 'inherit' | 'warning-triangle' | 'inherit-triangle' Adds a dot or triangle badge with a status color. The badges and their colors are shown in the following list:
  • 'info' - blue dot
  • 'success' - green dot
  • 'warning' - yellow dot
  • 'danger' - red dot
  • 'inherit' - dot inherits color of full icon glyph
  • 'warning-triangle' - yellow triangle
  • 'inherit-triangle' - triangle inherits color of full icon glyph

Icons have different accessibility requirements depending on the context in which they are used.

The three main contexts to consider for icon accessibility are listed below.

  • Inert Icons
  • Interactive Icons
  • Status/Indicator Icons

Inert Icons

By default, icons are inert, meaning that they don’t provide any information to screen readers and are not interactive. Without additional markup, an icon is purely cosmetic and doesn’t need additional accessibility markup.

Interactive Icons

When icons are used in buttons or links, you can also include descriptive text that briefly explains the icon’s meaning for accessibility.

If the icon is an interactive item that doesn’t include descriptive text, add an aria-label to describe the interaction.

Status or Indicator Icons

If an icon is used to show a status, display some other indicator, or is part of a larger body of text content, add an aria-label for the icon and the role="img" attribute.
