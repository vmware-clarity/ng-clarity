Types

Clarity has three navigation components: header, sub navigation, and vertical navigation. The following sections provide information about the possible navigation types.

Header + Sub Navigation

Because both patterns are horizontal, use this combination only when the secondary navigation does not warrant the space taken up by the vertical navigation.

When the header is the primary navigation and sub navigation is the secondary navigation combine the header component with a sub navigation using the DOM structure shown in the code examples.

Header + Vertical Navigation

This is a common pattern for two levels of navigation. The primary navigation is in the header, secondary is in the vertical navigation.

Header

When there are only a few locations for users to navigate, use a header as the sole navigation element between pages.

Vertical Navigation

When there are more links that can fit into the header or a sub navigation, consider using only a vertical navigation and combining the navigation into organized groups of locations.

Sub Navigation + Vertical Navigation

This combination can be a resolution to scenarios, such as those in the following list.

  • Your links don’t fit in the header and you need to use a sub navigation for primary navigation.
  • Your top-level locations have child navigation locations, so you combine sub navigation with vertical navigation elements specific to each top-level location.

Header + Sub Navigation + Vertical Navigation

This schema will not result in a clear visual hierarchy because of the adjacent horizontal patterns.

An alternative is to use the header and vertical navigation and then provide further navigation in the content area by using tabs.
