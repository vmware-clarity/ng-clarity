Usage

Stack views are designed for use in the main content area and modals. Use a stack view when you want to:

  • Display related key/value pairs, for example, an object’s settings.
  • Progressively disclose data. This is useful when you have a large set of key/value pairs or you want to enable users to reveal more complex or less frequently used data as needed.

Enabling Value Editing

In this pattern, the component includes an edit button on the top-right of the stack view. When a user clicks it, an editable stack view opens in a modal. This design prevents users from accidentally altering a value in the main content area. Common editing controls include input fields, select boxes, checkboxes, and radio buttons.

Labels

  • Provide terse labels that let users know what is under the label. Try to keep the label to one line.
  • Use noun phrases, sentence-style capitalization, and no ending punctuation.
  • Avoid using icons.

Highlights and Horizontal lines

The stack view highlight color (HSL: 198, 100, 96) differs from the usual Clarity highlight color to make it more distinct. Expanded sections have a lighter background color to make it easier to understand the hierarchy.
Horizontal lines in the stack editor are for readability–they help users quickly discern the relationship between columns.

Accessibility

To provide better accessibility to the stack view, some attributes must be set manually in the .stack-block-label element in the static form or use one of the provided inputs to ClrStackBlock.
