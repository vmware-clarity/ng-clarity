The stepper is an extension of the base accordion component. It structures multi-step processes into two or more expanding panels that break up complex workflows.

Usage

Use the stepper component:

  • When you have complex and long work processes that may not fit in one viewport
  • When users need to see the forms in-line with the rest of the page content
  • When users need more detail than step titles to understand the process

Check in the patterns which component and when is best to use for your multi-step process:

Anatomy and Behavior

  • Stepper panels are composed of two sections, the header and the input section.
  • The header contains a title and a description for the step.
  • When closed, the panels provide a high-level overview of the entire workflow at a glance.

Header

The step header section has three states: completed, active, to-do. When a step is in its active state, the header section is highlighted and its input section is visible for users to interact. As users complete steps, the steps are marked with a green bar to the left, and a summary of user's input will replace the description.

Input Section

The input section has two elements: the form and the buttons. The form is used to collect input from users to finish the step. The buttons section will allow the users to proceed to the next step or finish the workflow. To go back to a previous step, users click the step title of one of the previous steps. That step will then become the active step with its input section visible for editing.

Final Call to Action

For the button label in the final call to action, we recommend that the copy also describe an immediate outcome of completing the stepper. If the workflow will require more time to process after the stepper is complete, then FINISH or other non-specific terminology may work better.

Though we recommend action-oriented button labels, there are use cases where labels like DONE work well. If the user is viewing read-only information, we recommend using labels like DONE.

Code & Examples

The Stepper is an extension of the base Accordion component. The stepper requires use of the Angular Forms API. Using the Forms API allows the stepper to easily reflect the form state of each step in the stepper template.

Leverage our *clrIfExpanded structural directive on the clr-stepper-panel to only instantiate children when they are displayed.
