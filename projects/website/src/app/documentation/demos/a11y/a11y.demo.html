The @clr/addons/a11y package contains a number of components, directives, and services to improve accessibility within your application.

Usage

This package provides tools to handle common accessibility patterns such as managing tab overflow content, displaying required field legends in forms, and detecting browser zoom levels for responsive adjustments.

Features

Tabs Overflow Directive

Automatically include tab links in an overflow menu if there is not enough width to display them all. This is supported by default in Appfx Tabs, but requires the appfxOverflowTabs directive when using Clarity Tabs.

Required Field Legend

A generic required field legend to be used with forms. Consumers can use the TranslateService to sync the locale with their application locale.

Zoom Detection

The package includes both a directive and a service for detecting zoom levels based on document body size using ResizeObserver. This allows for layout adjustments at high zoom levels (200% and 400%).

You need to import the AppfxA11yModule in your module to be able to use the components and directives.

Add the directive appfxOverflowTabs to your <clr-tabs> component to activate automatic overflow detection.
