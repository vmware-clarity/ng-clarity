An accordion is a collection of vertically stacked sections with multiple content panels. It can deliver large amount of content in a small space through progressive disclosure.

Usage

A user can expand or minimize each panel within the accordion. Each panel’s header title provides a short, high-level overview of the panel content so the user can decide which sections to read. For example, you might design FAQ content using an accordion.

Accordions can make information processing and discovery more effective. However, it does hide content and makes it easier for users to not notice or read content that might be important. If all content is important to most users, don’t use an accordion. Instead use a full scrolling page with normal headers.

Content

Keep text as brief as possible. In the header title, concisely convey the nature of the content without overflow.

Behavior

To expand or collapse the accordion panels a user clicks the panel header. By default, only one panel can be open at a time. However, you can optionally implement accordions to allow multiple open panels.

Accordion States

Accordion panels can be expanded, disabled, or collapsed.

Code & Examples

Accordions by default only allow one panel to be open at a time. To enable multiple panels set clrAccordionMultiPanel="true" on the clr-accordion component.

Leverage our optional *clrIfExpanded structural directive on the clr-accordion-panel to only instantiate children when they are displayed.


The Accordion header should include only a title and brief info (if needed). Clickable elements are not allowed.
