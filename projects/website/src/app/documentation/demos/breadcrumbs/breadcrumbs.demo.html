Breadcrumbs provide information about a page's structural hierarchy. They contain links back to each previous page the user navigated through to get to the current page.

Usage

Use Breadcrumbs only when nested pages can't be avoided and there is no visual option from the navigation to lead the user back to the first nested page. Be advised that nested pages are not recommended best practice and try to evade using this approach.

Use Breadcrumbs when there are more than 3 nested pages, if less use the Back button.

The first link on the Breadcrumb should be the first actionable item from the vertical navigation. It could be on every level of the vertical navigation.

If there are more than 3 pages to show in the breadcrumb, use the collapsable instance of the component.

Anatomy

A breadcrumb consist of the following elements:

Interactive Item

Each item represents a single section in a hierarchical path. A clickable item contains a link to its corresponding page in the hierarchy.

Separator

Separates the breadcrumb items from each other.

Non Interactive Item / Current page

The current page is the last item of the breadcrumb. It is a not interactive item.

Back Button

Use a back button instead of using breadcrumbs, if there are less than 3 nested pages.

Back button is a Clarity Flat Compact button component

Placement

Breadcrumbs or the Back button should be placed in the header bar, above the page title.
