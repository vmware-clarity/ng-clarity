Cards are containers around a related grouping of content and actions.

Usage

Use a card to present high-level information and guide the user toward related actions and details. Clarity offers clickable cards if you want to use a card to initiate an action.

Typical elements included in cards include the following items:

  • Images
  • Dropdown menus
  • Media blocks
  • Lists and List groups

Anatomy

  • Cards can have dividers.
  • Flat buttons should be used on cards to keep their prominence in perspective with other information on the card.
  • Place the primary action and a single additional action, if required, in the card footer, left-aligned. This placement supports the F-pattern layout. For more than two actions, use a dropdown menu.
  • Do not place more than eight items in any dropdown menu.

Behavior

In similar card groups, consider enabling users to switch between card view and datagrid view. While cards show richer content than a datagrid, a datagrid lists more items at once.

Place the toggles for switching between views in the upper-right of the card group. The card group should be the default view.

Placement

Group cards by theme or element. Similar content helps scanning because users can quickly find and compare information of interest. Content of varying types often shows the bigger picture.

A grid places cards in fixed rows and columns. In this fixed design, you can often fit the cards into a smaller vertical space, which means less scrolling for the user. It's easy for users to quickly scan content in this layout. If in doubt, use the grid.

Cards can contain text, images, data visualizations, or multimedia. Ensure that the content serves your use case. Keep it simple and legible. Avoid using too much content, overloading the card with too many actions, or placing links within the content.

Assuming that you are using the Metropolis font family, the card should have a title using 20px Metropolis Medium and content using 14px Metropolis Regular.

Code & Examples

Basic Card

A .card can contain a .card-header, .card-footer, and one or more .card-blocks.

Clickable Cards

Adding the .clickable class makes the entire card clickable, initiating a single action.

Images in Cards

A .card-img can be placed anywhere in the card, or it can occupy the entire card.

Dropdowns in Cards

The footer can contain two actions. For more actions, use a dropdown.

Card Media Block

A .card-media-block combines a .card-media-image and .card-media-description. The description can contain a .card-media-title and a .card-media-text.

Alerts in Cards

Cards can contain alerts.

Lists in Cards

Cards can contain lists.

List Groups in Cards

Cards can contain list groups.

Progress Bars in Cards

Cards can contain progress bars.

Card Layout

Clarity recommends using cards in a grid or a CSS column layout.

Cards in a Grid

Cards in CSS Columns

Cards can be placed in .card-columns. The default number of columns is three, but can be changed to two or four by adding .card-columns-2 or .card-columns-4.

Clarity offers clickable Cards and Information Cards.

The Card header should include only a concise title with no clickable elements.

Information Card Footers can contain only Buttons or Button Groups.

Clarity supports only the following non-clickable elements: Status/Indicator Icons, Badges, and Labels.

Clickable cards must not contain any clickable elements.
