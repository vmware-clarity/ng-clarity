Custom Select All

By default, the "select all" checkbox in the datagrid header selects or deselects all currently displayed items using the built-in selection logic. However, in some scenarios you may need full control over what happens when the user clicks the "select all" checkbox. The clrDgCustomSelectAll feature lets you override the default behavior and implement your own select-all logic.

When to use

Custom select all is particularly useful in the following scenarios:

  • Virtual scroll datagrids — In a virtual scroll datagrid, only a subset of rows are rendered in the DOM at any given time. The default select-all behavior only toggles selection for rendered rows. With custom select all, you can select items from the entire dataset, including rows that are not currently rendered.
  • Server-driven selection — When your data is fetched from a server and you want the select-all action to trigger a server-side operation (e.g., marking all records as selected in the backend).
  • Conditional selection — When you want to apply custom filtering logic to determine which items should be selected (e.g., select only items matching certain criteria).

Basic usage

To enable custom select all, set [clrDgCustomSelectAllEnabled] to true on the <clr-datagrid> component and listen to the (clrDgCustomSelectAll) output event. When the user clicks the "select all" checkbox, the datagrid will emit a boolean value through this event instead of modifying the selection internally.

The emitted value is true when the checkbox is being checked (select all) and false when it is being unchecked (deselect all).

Usage with virtual scroll

Custom select all is especially important when using virtual scroll. In a virtual scroll datagrid, only the rows visible in the viewport (plus a small buffer) are rendered in the DOM.
