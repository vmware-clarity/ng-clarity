Datalist is a type of list that becomes contextualized through the selection of extensive pre-defined filters. Datalists can allow a custom input value. For example, if your users must select a country from a long list, you could provide pre-defined filters for each continent. You could also allow the user to start typing the name of the country to narrow down the options.

Usage

Use a datalist when the item list is long and you want to allow users to filter it to a manageable size and easily find the right selections. With a datalist, you can provide pre-defined filter options and allow users to input their own filter value.

Do not confuse a datalist element with the select input. Datalist elements usually have more than 13 options that the user will need to filter or narrow down before making a choice. If there are 3-13 options, consider using the select input.

@for (item of items; track item) { } Helper message
Use a datalist:
  • When the user needs to choose from a long list of items to be filtered before selection

States

Enabled

Use when the datalist is interactive.

@for (item of items; track item) { } Helper message There was an error

Active

Use when a datalist item is selected by a user using an input method, such as a mouse or the keyboard.

@for (item of items; track item) { } Helper message There was an error

Disabled

Use when the datalist is not interactive. For example, when a user does not have permission to change specific settings.

@for (item of items; track item) { } disabled message

Error

Use to communicate a user or system error. For example, this state can happen in the following scenarios:

  • When the user enters an invalid character
  • During validation if the user moves away from the datalist without making a selection
  • When the user attempts to submit a form containing a datalist with no selection.

Success

Use to provide real-time feedback immediately after a user’s valid input.

Code & Examples

The HTML datalist element contains a list of pre-defined option elements that can be chosen for an input control value. Or, the user can type a different value into the input.

Natively, the id for a datalist provides the link to an input via the value given for the input's list attribute. Clarity handles this association by default; it generates and associates the datalist id with its corresponding input element via the list attribute on the input. The app marks the input for a datalist by using the clrDatalistInput directive on the input and default, unique values are generated for the component. If there is an application side need to declare a custom id for the datalist, this value will be used instead of the generated id.

Validation

Datalists in template driven forms implement the same validation attributes that are described by native HTML form validation. Datalists in reactive forms allow apps to utilize the built in Angular validators (e.g required or minlength) or, they can add custom validator functions directly to the form control model in the component class.

As with other Clarity form controls, the datalist and its associated input belong inside a container. In this case it is named clr-datalist-container. Besides the required input with the clrDatalistInput directive and the datalist element this container takes three other optional elements. An optional label element, clr-control-helper and the clr-control-error elements if there is validation for it.
