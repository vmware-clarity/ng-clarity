clrStartDate and clrEndDate Directive

To use the date range picker, add clrStartDate and clrEndDate directive to the start and end date input fields respectively. Then, place the input inside the clr-date-range-container container element.

It is recommended that you set the type to date as it increases readability of your code, helps with SEO, and keeps your HTML markup semantic. When the date picker is enabled, this type is overridden to text to disable the built-in date pickers that some desktop browsers provide.

Min/Max attributes

The earliest and latest acceptable dates can also be set. Just like the native HTML5 date spec a yyyy-mm-dd string can be used in the value for min or max. If the max value isn't a possible date then that the input has no max value. Similar for min values, if the string used is not a possible date then the input will not have a min value.

Only dates inside the min/max range will be selectable for the input when there is a bound placed for the min or max attribute.

Predefined Range Options

"Predefined Range Options" refers to a set of readily available time periods (like "Today", "This Week", "Last Month", "This Year") that users can select from when choosing a date range in a system, rather than having to manually input a start and end date each time. Essentially, a quick way to access common date ranges without needing to customize them every time. Range options is an array of object with label and value properties. Label refers to time periods (like "Today", "This Week") and value will be array of date objects with start and end values.
