Multi-Page Dialog

The multi-page dialog (appfx-dialog) is a Clarity modal that hosts tabbed workflow steps. Each step is rendered by a separate component registered via the Step model.

Import

Usage

Wrap steps inside appfx-dialog. Pass the steps array and a workflow model. Optionally supply a closeHandler to perform async submit/cancel logic.

AppfxMultiPageDialogModule

The main module to import. Exports: appfx-dialog, appfx-dialog-header, appfx-toggle.

DialogComponent appfx-dialog

Input Type Default Description
title string — Modal title
subTitle string | Observable<string> — Secondary title displayed next to the primary title with a separator
steps Step[] — Array of step definitions
model WorkflowModel — Workflow model shared across steps
tabLayout TabLayout horizontal Layout direction of tabs
size ModalSize — Clarity modal size: sm, md, lg, xl, full-screen
defaultButton 'submit' | 'close' close Which footer button is highlighted
height string 545px min Explicit dialog height (e.g. '800px')
loading boolean false Show spinner overlay
opened boolean true Controls dialog open/close state
closeHandler CloseHandler — Object with onSubmit / onCancel callbacks returning Observable
okButtonLabel string OK Submit button label
cancelButtonLabel string Cancel Cancel button label
showTabLinks boolean true Show or hide tab navigation links
disableTabsContent boolean — Disable tab content interaction
Output Type Description
onModelChange EventEmitter<ModelChange[]> Emits when workflow model variables change
onClose EventEmitter<void> Emits when dialog is closed
openedChange EventEmitter<boolean> Two-way binding partner for opened

ToggleComponent appfx-toggle

A labeled checkbox toggle for use inside appfx-dialog-header.

Input Type Description
label string Label text
toggleId string Input element ID (auto-generated if omitted)
toggleState boolean Current toggle value (two-way bindable)
disabled boolean Disables the control
