cds-layout is a CSS attribute utility for building flexible, consistent layouts using Clarity's spacing tokens. It provides a declarative API for controlling flex direction, alignment, gaps, padding, margin, and container widths directly in HTML — without writing custom CSS.

Usage

Add the cds-layout attribute to any HTML element. The attribute value is a space-separated list of tokens. No import or module registration is required — the styles are applied via the global Clarity stylesheet.

Layout Types

Horizontal Layout

A horizontal layout arranges children in a row using flex. Items wrap to the next row by default when they overflow the container.

Padding & Margin

Padding and margin shorthand tokens follow the same size scale as gap. All accept the same size values: none | xxs | xs | sm | md | lg | xl | xxl.

Token Applies to
p:md All four sides
p-x:md Left and right (inline axis)
p-y:md Top and bottom (block axis)
p-t:md Top only
p-r:md Right only
p-b:md Bottom only
p-l:md Left only
m:md   m-x:md … Same pattern applies for margin

Container Widths

Use container:<size> to constrain the max-width of a section to a standard breakpoint. Add container:center to center it horizontally.

Token Max-width
container:sm 576px
container:md 768px
container:lg 992px
container:xl 1200px
container:center Centers the container using margin-inline: auto

Wrap Control

By default, a horizontal layout wraps items to the next row when the container overflows. Use wrap:none to disable wrapping (e.g., for toolbars).
