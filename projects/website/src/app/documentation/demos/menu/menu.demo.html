Menu

The context menu (appfx-menu) provides a right-click style contextual menu built on top of Clarity's clr-dropdown. It supports nested submenus, icons, keyboard navigation, and 200%/400% zoom accessibility.

Import

Setup

Declare <appfx-menu> and add exactly one <menu-outlet> to your root layout. Call menu.show(event, x, y) to position and open it. The menu content (actions, headers, separators, submenus) must be declared statically — use [enabled]="false" to disable items instead of adding/removing them.

AppfxMenuModule

Exports: appfx-menu, appfx-menu-action, appfx-menu-header, appfx-menu-separator, menu-outlet.

MenuComponent <appfx-menu>

Member Type Description
show(e, x, y, trigger?) void Opens menu at given viewport coordinates. Optionally pass a trigger element to restore focus on close.
close(e) void Closes the menu
text string When used as a nested menu, sets the submenu label
opened EventEmitter Emits once when the menu opens
closed EventEmitter Emits once when the menu closes

MenuActionComponent <appfx-menu-action>

Input Type Description
text string Action label
iconClass string Optional icon CSS class
shortcut string Keyboard shortcut hint (right-aligned)
enabled boolean Default: true. Disabled items render but are unclickable.
handle EventEmitter Emits when the action is clicked while enabled

Important: Static item constraint

The set of menu items must be declared statically — the menu will throw if items are added or removed after initial render. Use [enabled]="false" to disable items asynchronously instead.
