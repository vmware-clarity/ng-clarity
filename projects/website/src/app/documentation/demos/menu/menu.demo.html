appfx-menu provides a reusable contextual menu built on top of Clarity's clr-dropdown. On top of the generic clr-dropdown, it adds:

  • positioning based on an anchor point instead of an anchor button
  • positioning and scrolling that account for an encompassing container (typically the viewport)
  • a domain-specific declarative interface instead of custom HTML markup
  • support for 200% and 400% zoom

Anchor point

clr-dropdown supports displaying a menu anchored to a button but doesn't support the pattern of showing it as a contextual menu. appfx-menu lets you show a menu at specific coordinates without an anchor button, via menu.show(event, x, y).

Restricting the overflow

It's typical to position a menu somewhere other than next to the anchor point when it would otherwise overflow the viewport. This is handled similarly to what you'd see in a desktop application — when a menu positioned below its anchor point would overflow:

  • if it overflows horizontally, it's shifted left
  • if it overflows vertically, it's shifted up
  • if it still overflows after being shifted to the top of the viewport, a scrollbar is displayed
  • if a submenu overflows horizontally, it's flipped to the left

Domain-specific declarative interface

Menu content is composed declaratively out of:

  • headers
  • separators
  • nested submenus
  • actions with text, icon, and shortcut

Setup

Declare <appfx-menu> and add exactly one <menu-outlet> to your root layout. Call menu.show(event, x, y) to position and open it. The menu content (actions, headers, separators, submenus) must be declared statically — use [enabled]="false" to disable items instead of adding/removing them.

AppfxMenuModule

Exports: appfx-menu, appfx-menu-action, appfx-menu-header, appfx-menu-separator, menu-outlet.

MenuComponent <appfx-menu>

Member Type Description
show(e, x, y, trigger?) void Opens menu at given viewport coordinates. Optionally pass a trigger element to restore focus on close.
close(e) void Closes the menu
text string When used as a nested menu, sets the submenu label
opened EventEmitter Emits once when the menu opens
closed EventEmitter Emits once when the menu closes

MenuActionComponent <appfx-menu-action>

Input Type Description
text string Action label
iconClass string Optional icon CSS class
shortcut string Keyboard shortcut hint (right-aligned)
enabled boolean Default: true. Disabled items render but are unclickable.
handle EventEmitter Emits when the action is clicked while enabled

MenuHeaderComponent <appfx-menu-header>

Input Type Description
text string Header label
iconClass string Optional icon CSS class

MenuSeparatorComponent <appfx-menu-separator>

Renders a horizontal divider between menu items. No inputs.

Important: Static item constraint

The set of menu items must be declared statically — the menu will throw if items are added or removed after initial render. Use [enabled]="false" to disable items asynchronously instead.
