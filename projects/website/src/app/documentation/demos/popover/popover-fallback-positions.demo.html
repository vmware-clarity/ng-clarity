The availablePositions array defines fallback positions that are tried in order when the preferred position does not fit the viewport. The CDK checks each position sequentially and uses the first one that fits.

Resize the browser or scroll the page so the button is near the edge. The popover prefers top-right, but falls back through bottom, top, left, and right positions.

Fallback Positions

Preferred: top-right
Fallbacks (in order): bottom-middle, bottom-left, bottom-right, top-middle, left-middle, right-middle

Move the button near an edge of the viewport to see the overlay reposition automatically.
