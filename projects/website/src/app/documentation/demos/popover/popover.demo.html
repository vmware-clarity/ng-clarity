A popover is a temporary overlay anchored to an element on the page. It displays supplementary content such as details, confirmations, or custom controls without navigating away from the current context.

Usage

The Popover utility is the low-level foundation behind Clarity's overlay components. It is built on the Angular CDK Overlay, which handles rendering, positioning, and z-indexing. Use the popover utility directly when you need custom overlay behavior that doesn't fit the standard components:

  • A confirmation dialog anchored to a delete button, asking the user to confirm an action before it proceeds.
  • A detail panel showing extra metadata when the user clicks a table row or list item.
  • A custom picker such as a color or icon selector attached to an input field.

For common patterns, prefer the specialized components built on top of popover:

Scenario Recommended
Action menu with items, headers, dividers Use Dropdown
Contextual help with an arrow and close button Use Signpost
Brief, non-interactive description on hover Use Tooltip

Placement

Element Origin

The popover is positioned relative to its origin element using a connected position strategy. You choose a preferred position — for example, below and to the left of the origin. If there isn't enough room in the viewport for that position, the overlay automatically tries a set of fallback positions until it finds one that fits.

Twelve positions are available, covering every combination of direction (top, bottom, left, right) and alignment (start, middle, end). You can also provide custom fallback positions using CDK's ConnectedPosition interface for full control over how the overlay repositions itself.

Point-based Origin (Context Menus)

Popovers can also be opened at an arbitrary { x, y } screen coordinate instead of an element. This is useful for context menus triggered by right-click, where the overlay should appear at the cursor position rather than anchored to a specific element.

Call openAtPoint({ x, y }) on the ClrPopoverService to open the popover at a specific point. Point-based origins automatically close on any scroll event and delay outside-click detection by 500 ms to prevent the initial right-click mouseup from immediately dismissing the popover.

Behavior

Clicking the trigger element opens the popover. It stays visible until the user explicitly closes it or interacts elsewhere:

  • Clicking the trigger again toggles the popover closed.
  • Clicking outside the overlay dismisses it (configurable).
  • Pressing Escape closes the popover.

Scroll Behavior

Popovers support two scroll strategies, inspired by the CDK Overlay scroll strategies:

  • Reposition (default) — the overlay follows the anchor as the page scrolls, staying aligned with it. This is appropriate when the popover should remain visible during scrolling.
  • Close on scroll — the overlay is dismissed as soon as any ancestor container scrolls. This is appropriate for popovers attached to items in scrollable lists or tables, where the anchor may scroll out of view.

Focus Management

When the popover closes, focus returns to the origin element to maintain keyboard accessibility. While the popover is open, focus should be trapped inside the overlay so keyboard users cannot tab into the page behind it. For point-based origins (no element), focus restoration is skipped since there is no trigger element to return to.

Content

  • Keep popover content concise. If you need a full page of content, consider a modal or side panel instead.
  • Include a close button inside the overlay so keyboard and assistive-technology users can dismiss it.
  • Popovers accept any HTML content — text, forms, images, or other components.
  • Avoid nesting popovers inside other popovers. If the content requires deeper navigation, use a different pattern.

Accessibility

  • Give the overlay a role="dialog" attribute so screen readers announce it as a dialog.
  • Use cdkTrapFocus on the content container to trap keyboard navigation within the overlay while it is open.
  • Ensure the trigger element is a focusable, interactive element (typically a button).

Setup

The popover utility is built on the Angular CDK Overlay. It abstracts the CDK into a set of directives coordinated by a transient service. To use it in your component:

  1. Import the module: Add ClrPopoverModuleNext to your component's imports array. This provides all popover directives ([clrPopoverOrigin], [clrPopoverOpenCloseButton], [clrPopoverCloseButton], and *clrPopoverContent).
  2. Provide the service: Add ClrPopoverService to your component's providers array. Each popover needs its own service instance — if you have multiple popovers, each host component must provide its own.
  3. Add a boolean property (e.g. open = false) to track the popover state in your template.

Fallback Positions

Use the availablePositions input to provide custom fallback positions. Each fallback is a CDK ConnectedPosition object that defines how the overlay attaches to the anchor. When the preferred position doesn't fit within the viewport, the overlay tries each fallback in order until one fits. See the CDK FlexibleConnectedPositionStrategy documentation for details on the ConnectedPosition interface.

Scroll to Close

By default, scrolling repositions the overlay to keep it aligned with the anchor. Set scrollToClose to true to dismiss the popover instead. This is useful for popovers attached to items in scrollable lists or tables where the anchor may scroll out of view.

Context Menu (Point-based Origin)

Open a popover at an arbitrary screen coordinate using ClrPopoverService.openAtPoint(). Right-click inside the dashed area to see the popover appear at the cursor position. No [clrPopoverOrigin] directive is needed — the origin is set programmatically to the click coordinates.

API Reference

The popover utility builds on the Angular CDK Overlay. The types ConnectedPosition, FlexibleConnectedPositionStrategyOrigin, and OverlayRef referenced below are from the @angular/cdk/overlay package.

ClrPopoverService — service

A transient service provided at the component level. Each popover instance gets its own service to manage state independently.

Type: boolean

Type: Observable<boolean>
 Observable<boolean> Emits whenever the open state changes.

Type: FlexibleConnectedPositionStrategyOrigin
 FlexibleConnectedPositionStrategyOrigin The positioning origin for the overlay. Can be an ElementRef (set automatically by [clrPopoverOrigin]) or a ClrPopoverPoint ({ x, y }) for point-based positioning.
originElement
Type: ElementRef | null
 ElementRef<HTMLElement> | null Returns the origin as an ElementRef when element-based, or null for point-based origins.

Type: ClrPopoverPoint | null
 ClrPopoverPoint | null Returns the origin as a { x, y } point when point-based, or null for element-based origins.

Type: Method
 void Toggles the open state and stores the triggering event. Used internally by [clrPopoverOpenCloseButton].

Type: Method

Type: Method
 void Returns focus to the origin element. Called automatically when the popover closes. No-op for point-based origins.

ClrPopoverPoint — interface

A simple interface representing a screen coordinate: { x: number; y: number }. Used with openAtPoint() to position the popover at an arbitrary point.

*clrPopoverContent — structural directive

Creates and manages the CDK overlay. The content is only present in the DOM when the popover is open. Under the hood, it uses a FlexibleConnectedPositionStrategy and a DomPortal to attach content to the overlay container.

[clrPopoverOrigin] — attribute directive

Marks the element as the positioning reference point for the overlay. Registers the element's ElementRef with the ClrPopoverService, which the CDK FlexibleConnectedPositionStrategy uses as the origin for positioning calculations.

[clrPopoverOpenCloseButton] — attribute directive

Place on a button to toggle the popover open and closed. Calls toggleWithEvent() on click and handles keyboard events (Enter, Space) for accessibility.

Note: [clrPopoverOrigin] and [clrPopoverOpenCloseButton] are independent directives and do not have to be on the same element. However, placing them together on the trigger button is the recommended pattern — it ensures the overlay positions relative to the same element the user clicked. Separating them across different elements is possible but requires careful consideration of positioning and focus behavior.

[clrPopoverCloseButton] — attribute directive

Place inside the overlay content to provide a close action. Closes the popover and restores focus to the origin element. Only valid inside a *clrPopoverContent container.

POPOVER_HOST_ORIGIN — InjectionToken

An injection token for providing the host element as the popover origin. Used internally by ClrPopoverHostDirective to register the host's ElementRef as the origin for components like dropdown, signpost, and tooltip.
