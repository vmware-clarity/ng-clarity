@clr/addons/property-view renders structured key-value data organised into categories and expandable sections. Used by the Wizard summary page and Stepper step descriptions, but also available as a standalone component for detail or summary panels.

Overview & Rationale

Property View provides a way for rendering multiple properties (key-value pairs) in a hierarchical manner.

Design

The properties hierarchy consists of the following nodes:

  • Root node — root of the hierarchy, contains Category nodes.
  • Category node — logical grouping of properties, contains Section nodes.
  • Section node — logical grouping of properties, contains Property and Message nodes.
  • Property node — a key/value pair, contains a Property Key node and Property Value nodes.
  • Property key node — property key with text and optional icon.
  • Property value node — property value with text and optional icon.
  • Message node — message with text and optional icon.
