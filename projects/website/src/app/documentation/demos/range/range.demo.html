The range input component consists of a slider that can be moved to represent some value between the minimum and maximum values. It allows the user to select a value when the precise value chosen is not considered necessary.

Usage

Use the range component when it is acceptable to have an imprecise input control and the exact value is not important. Range controls work well when users are more concerned with estimated percent distance between the minimum and the maximum values.

See more usage guidelines in our patterns:

States

Enabled

Use when interactive and ready for input.

Disabled

Use when not interactive. For example, when a user does not have permissions to edit the range.

Error

Use to communicate a user or system error. This validation can happen when the user moves away from the input or after attempting to submit a form.

Success

Use to provide real-time feedback immediately after a user’s input. For example, the input was acknowledged by the application.

The Range input uses standard HTML5 attributes and can be used with Clarity forms.

The Range Input component includes a native input type range form element (with all native limitations), a label, helper text, and status messaging.

Avoid nesting components within helper text or error messages, as this can cause overloading. Consult your design team for guidance.
