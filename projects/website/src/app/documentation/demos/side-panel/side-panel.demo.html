Side panels display supplementary contextual actions and information.

Usage

Side panel component can be used similarly to a modal that slides over the primary content. Use it for additional information or actions that complement the main content on the page.

Position

By default the side panels are positioned on the right side of the document body.

But using the clrModalHost directive, they can be positioned inside an arbitrary container, that only occupies a portion of the screen.

Pinning

The side panel has a pinning functionality, which gives the user the capability to "freeze" the panel. This is achieved through the clrSidePanelPinnable input. Currently, the side panel pinning also depends on the clrSidePanelSize input to work properly. When pinned, all normal ways to close the panel are blocked and it's displayed side by side with the content, instead of overlapping it. A pinnable container can also be inline.

Sizes

There are multiple sizes for side panels. The default size is medium, however other sizes are available. Select the size best suited for the content and screen size displaying the side panel. Remember to test responsiveness.

Behavior

Consider using side panels for the following scenarios: alerting the user, confirmation dialog boxes, and task-oriented workflows.

Dismissing Side Panels

There are different ways to dismiss a side panel but a user needs to intentionally make that choice. A side panel should not disappear on its own.

Provide options using action buttons to either cancel or proceed or a close button that should eventually dismiss the side panel.

Stacking Side Panels

Side Panels should not launch other side panels. Stacking side panels makes it hard to dismiss them and confuses the user on their levels of importance and context.

Instead of stacking, seek alternatives such as inline expansion and inline task completion within the side panel.

Accessibility and managing focus

The VMware Accessibility team recommends forcing focus to the title of a side panel when it is first opened.

Note that the clr-side-panel component automatically focuses the side panel title.

Side Panel Component

Inline

The side-panel can be displayed inline in an arbitrary container instead of on the page body. This is done through the clrModalHost directive on the desired container.

Pinning

The side-panel can be pinned to the container.

The side-panel pinning can be pinned by default with no pin/unpin buttons.

Sizes

The component offers different sizes through the clrSidePanelSize input. The default, empty size is medium.

Static Backdrop

If you want the side panel not to be closable with a click on the backdrop, use the clrSidePanelStaticBackdrop. If true, clicks on the background will not close the side panel. The default value is false.

Alternate Close

If you want to intercept the side panel close event, use clrSidePanelPreventClose and clrSidePanelAlternateClose. This can be used to request confirmation on close.

Style

The Side Panel component is based on the Modal, so it can be customized through the Modal CSS Custom Properties and CSS Classes.
