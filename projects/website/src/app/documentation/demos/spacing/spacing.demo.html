Clarity provides space tokens, making it easier to create rhythmic and consistent designs throughout your product. Included are two types of space tokens: layout space for page layout and component space for use within components.

Layout Space

Layout space is the smaller set in T-shirt sizes e.g. XXS and is based on a 4px proportional system. This set is very useful for designing screens with space between components that define a consistent vertical rhythm. Since margin and padding attributes are absent from Clarity components, you can create designs using a bricks (components) and mortar (space) metaphor. By specifying the use of space in this way, combined with our Layout Utilities, the code and design have a 1:1 correspondence - no surprises and easy to finesse.

@for (token of layoutSpaceTokens; track token) { }
{{ token.name }}
{{ token.pxValue / 20 }} rem ({{ token.pxValue }}px)
Pixel values based on 1 rem = 20px

Space tokens used vertically and horizontally in a screen layout.

Component Space

Component space consists of finer increments of adjustment necessary for spacing within components. As with layout space, pixel sizes are provided to assist in understanding them in a 100% magnification view, but under the hood they refer to scalable values allowing for magnification and responsive design. In most cases designers will be using only layout spacing. Component space should be used primarily for designing components. Keep this in mind if you need something Clarity does not provide. Using the Space tokens to build such a component will make it behave as expected with regard to theming.

@for (token of componentSpaceTokens; track token) { }
{{ token.name }}
{{ token.pxValue / 20 }} rem ({{ token.pxValue }}px)
Pixel values based on 1 rem = 20px

Space tokens used vertically and horizontally within button and card components.

Layout Utilities

By using Space tokens, you can achieve flexibility and dynamic behavior in your screen designs. For example, lets say we have a group of elements arranged vertically and we wish to fine tune the visual rhythm and create an effective information hierarchy. In this example we’ve decided that we want XXL between the top of the page and the group of buttons, MD between the buttons and the horizontal rule, XS between the rule and the title, and LG between the title and the body copy. We specify each using the T-shirt size scale. The layout space tokens will be used to group and arrange the components. In this case layout will specify vertical and left alignment. We also have specified that the button group is left aligned and uses a MD space between each button.

  1. XXL top and side margins.
  2. Horizontal group. Left-aligned with MD space between elements.
  3. Vertical group. Top-aligned with varying space between elements.

This is a very simple example. The utilities have numerous options for specifying not only the alignment and gap attributes of a group, but also how a component scales, wraps, stretches, responds (or doesn’t) within its container. Refer to the layout and spacing Storybook pages to view the full list of options.
