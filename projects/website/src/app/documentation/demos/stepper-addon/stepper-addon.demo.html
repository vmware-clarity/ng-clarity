@clr/addons/stepper provides an inline multi-step form built on Clarity Stepper. It shares the same workflow engine as @clr/addons/wizard — the same Step descriptors and WorkflowModel can be reused across both components.

Wizard vs Stepper

Feature Wizard Stepper
Layout Modal dialog Inline page
Navigation Previous / Next buttons Collapse / expand panels
Step summary Separate summary page Below each panel header
Shared step API Same Step descriptors and WorkflowModel

Import AppfxStepperModule and AppfxWorkflowCoreModule.

StepperComponent Inputs / Outputs

Name Input/Output Type Required Description
steps Input Step[] Yes Ordered step descriptors — title, component class, model, etc.
wizardModel Input WorkflowModel Yes Shared model flowing between steps
loading Input boolean No Externally force loading state
showBackButton Input boolean No If set to true, a Back button is displayed on every step except the first visible one.
showCancelButton Input boolean No If set to true, a Cancel button is displayed on all steps.
usePrimaryNextButton Input boolean No Use solid primary style for the Next/Finish button
onModelChange Output EventEmitter<ModelChange[]> No Emits when the workflow model changes
onFinish Output EventEmitter<void> No Emitted when the user submits the last step
onCancel Output EventEmitter<void> No Emits when the user clicks the Cancel button

Step

The Step interface describes each step of the stepper.

Property Type Required Description
title string Yes Step title
description string No Step description, rendered on the right of the step title
componentClass Type<any> Yes Step component class to render
model StepModel | StepModelFactory No StepModel — data model of the step, containing its input and output properties. StepModelFactory — factory function that returns StepModel instances; required if recreateComponent is set.
instantiateLazy boolean No If true, the step component is instantiated the first time it's activated. If false, it's instantiated immediately when the stepper is created. Default: false.
isRelevant Var<boolean> No Conditionally include the step in the workflow
mappings Mappings<any, any> No Property name mappings from the step model to the workflow model, if names differ
summary (builder: PropertyViewSectionBuilder, stepModel?: StepModel) => PropertyViewSectionModel No Invoked when the user moves to the next step. The result replaces the step's description, shown on the right of the step title.
recreateComponent (stepModelChanges?: ModelChanges) => boolean No Recreate the step component if the function returns true
