Tabs

@clr/addons/tabs renders a composition of step components as a tabbed interface using Clarity's clr-tabs. The workflow is orchestrated by the same WorkflowModelManager used by the Wizard and Stepper, so step components and step models can be reused across all three patterns without modification.

Tabs Structure

AppFx Tabs is built on top of Clarity Tabs and shares the same structural building blocks as Wizard and Stepper:

  1. Tabs component — hosts <appfx-tabs>, configures tabs and model, and exposes validate$() for the caller to invoke on save.
  2. Tabs model — a POJO with Var<T> properties shared across steps, identical in shape to a Wizard or Stepper model.
  3. Step component (per tab) — implements StepModelHolder to receive the typed step model, same as a Wizard page.
  4. Step model (per tab) — typed POJO with input/output properties, reusable between Tabs, Wizard, and Stepper.

Features

Horizontal, Vertical, or Secondary tabs

AppFx Tabs offers the flexibility to display tabs horizontally (the default), in a vertical orientation, or as a secondary tab in a horizontal layout. To customize the layout, set the tabLayout property.

Disable all tabs

Tab content can be disabled with the disableTabsContent property. The user will still be able to click around the tabs but will not be able to interact with the step components. When tab content is disabled, steps will not get validated.

Reusable in AppFx Dialog

AppFx Tabs is fully compatible with AppFx Dialog, with both components sharing the same underlying workflow functionality. The key distinction lies in how they are rendered, with AppFx Dialog appearing as a modal popup. This compatibility allows one to effortlessly use AppFx Tabs within AppFx Dialog, reducing code duplication and enhancing maintainability.

  • Collapsible tab links — auto-collapses at high zoom levels (accessibility)
  • Per-tab validation with error badges on invalid tabs
  • Programmatic validation via validate$() and validateActiveTab$()
  • Auto-focuses the first invalid tab when validate$() fails

AppfxTabsComponent (appfx-tabs)

Property I/O Type Default Description
tabs Input Step[] — Step definitions to display as tabs.
model Input WorkflowModel — Shared model injected across steps.
tabLayout Input TabLayout horizontal Layout: horizontal, vertical, or secondary.
disableTabsContent Input boolean false Disables all tab content interaction without hiding tabs.
loading Input boolean false Shows a loading indicator on the component host.
showLoadingIndicator Input boolean true Controls whether a loading indicator appears during validation.
showTabLinks Input boolean true Show or hide the tab navigation links.
autoCollapseTabLinks Input boolean true Auto-collapses vertical tab links at high zoom levels.
onModelChange Output EventEmitter<ModelChange[]> — Emits on any workflow state variable change.
activeTabChange Output EventEmitter<Step> — Emits the activated step when the current tab changes.
tabLinksOpenedChange Output EventEmitter<boolean> — Emits when the collapsible tab links panel opens or closes.

Step Interface

The Step interface describes the content of each tab.

Property Type Required? Description
title string Yes Tab title.
componentClass Type<any> Yes Step component class rendered inside the tab.
model StepModel | StepModelFactory No Data model of the step, containing input and output properties. Use a StepModelFactory if recreateComponent is set.
instantiateLazy boolean No If true, the step component is instantiated upon first activation. If false, it's instantiated immediately when the tabs component is created. Default: false.
mappings Mappings No Property name mappings between the step model and the tabs model, if names differ.
recreateComponent (stepModelChanges?: ModelChanges) => boolean No Recreate the step component whenever this function returns true.

Public methods

Method Returns Description
validate$() Observable<boolean> Validates all tabs; navigates to the first invalid one.
validateActiveTab$() Observable<boolean> Validates only the currently active tab.
isTabActive(step) boolean Returns true if the given step is currently active.
isReady boolean True when all visible tabs have completed their setup.
isLoading boolean True when any step model or the component itself is loading.
