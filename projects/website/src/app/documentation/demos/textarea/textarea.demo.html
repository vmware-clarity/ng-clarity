The textarea component provides a form control for long-form text input. When anticipating user inputs of a sentence or less, use the input component. Multiple usages of the input component strung together may constitute a form. See Forms for more details.

Usage

Use textarea when the expected user input may be more than a single line.

See more usage guidelines in our patterns:

States

Enabled

Use when interactive and ready for input.

Active

Use when selected by a user with an input method, such as mouse or keyboard.

Disabled

Use when not interactive. For example, when a user does not have permissions to edit the input.

Error

Use to communicate a user or system error. This validation can happen when the user moves away from the input or after attempting to submit a form.

Success

Use to provide real-time feedback immediately after a user’s input.
