ThemeBuilderComponent is a standalone, embeddable UI for customizing Clarity's color tokens (primary, info, success, warning, danger) across the light and dark theme, previewing the result live, and exporting a CSS override file.

Design

Inputs & Outputs

  • [presets] — the list of named color presets shown in the preset selector. Defaults to Clarity's built-in preset list (PRESETS), exported from @clr/addons/theme-builder so consumers can extend it (e.g. [...PRESETS, myPreset]) instead of replacing it outright.
  • (generatedCSS) — emits the generated CSS override text (a string) every time a color, preset, reset, or the warning-text-override toggle changes.
  • [customContent] — set to true to project your own preview through content projection instead of the built-in preview. The projected content still renders inside the themed preview sandbox, so it reacts to the light/dark toggle and every color edit exactly like the built-in preview does.

Limitations

  • Persisting the generated CSS is the consumer's responsibility — display, copy, or save the value emitted through generatedCSS.
  • Only the five status/primary aliases (primary, info, success, warning, danger) and their computed tint/shade/dark variants can be edited.
