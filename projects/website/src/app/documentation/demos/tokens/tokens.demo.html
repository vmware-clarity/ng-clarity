What Are Design Tokens?

Design tokens are named values for hundreds of CSS attributes that cover colors, fonts, spacing, and more, allowing you to maintain consistency across your application. By using design tokens, you can quickly adjust styles, experiment with different themes, and stay on brand. Whether you’re updating a color to align to brand standards or adjusting interactive cues, design tokens are the key to quickly updating user interfaces within the Clarity Design System.

Benefits

  • Consistency: Design tokens ensure uniformity across your application, maintaining a cohesive design language.
  • Maintainability: By centralizing design decisions into tokens, updates and modifications become more manageable and less prone to errors.
  • Theme-Readiness: Design tokens prepare your application for theming, allowing for easy adaptation to different visual styles and branding requirements.
  • Streamlined Workflow: Leveraging design tokens in both design and development phases simplifies the workflow, reducing duplication of effort and ensuring alignment between design and code.
  • Future-Proofing: Design tokens facilitate adaptation to future updates and enhancements, allowing your application to evolve with changing requirements and trends.

Within Figma, our Clarity Design System offers two types of design tokens: Global and Alias.

Benefits

  • Color Palette: Use Clarity Design System's color tokens when defining your color palette. Replace raw color values with token references to ensure consistency.
  • Spacing: Utilize layout and spacing tokens to maintain consistent spacing between elements. Tokens like --cds-global-layout-space-sm make it easy to achieve consistent margins and padding.
  • Typography: Apply typography tokens for font sizes, line heights, and font families. Use token references instead of hard-coded values.

Incorporating Tokens

To incorporate design tokens, please ensure your project is updated to Clarity Version 17 or higher*. 

*Please note that while designers typically access tokens through the Clarity components library, developers may need to update to ensure compatibility and consistency between design and code.

Globals

Global tokens begin with the string cds-global and are used to define various system values. They serve as the backbone of the token system, establishing fundamental values that remain consistent regardless of their usage within the system.

Benefits

  • Enhanced Design Consistency: Design tokens ensure that colors, typography, spacing, and other visual elements remain consistent across the entire application, helping designers maintain a cohesive and professional design aesthetic.
  • Efficient Theme Customization: Design tokens streamline the process of theme customization, allowing designers to experiment with different color schemes and styles more efficiently. This enables rapid iteration and exploration during the design phase, ultimately leading to more polished and visually appealing interfaces.

Aliases

Aliases play a pivotal role in the Clarity Design System, serving as a bridge between broad, universal values and specific, meaningful applications. By translating global tokens into context-specific customization, aliases contribute to intuitive design experiences that ensure consistency across components and themes.

For instance, the --cds-alias-status-info token is mapped to var(--cds-global-color-blue-700) in the light theme and var(--cds-global-color-blue-400) in the dark theme. This means that when you design with these tokens, your components will automatically adapt to different themes without needing separate designs for each.

Benefits

  • Enhanced Component Interactions: Aliases facilitate seamless customization of component interactions, allowing designers to fine-tune specific attributes.
  • Adaptive Theming: With aliases, adapting to different themes becomes easy, enabling designers to maintain design consistency while accommodating diverse visual styles.

  • CSS Variables: When writing CSS styles, reference design tokens using CSS variables. Replace fixed values with token references to ensure your styles align with the design system.

  • Theme Switching: Leverage the power of tokens for theme switching. Tokens can be remapped to different global values for dark and light themes, making your application theme-ready with minimal effort.

Aliases

Design token aliases serve as meaningful connections to global values within the Clarity Design System, enabling developers to achieve consistent styling and theme adaptability across components. With predefined mappings and streamlined referencing, alias tokens facilitate efficient development and maintenance practices, empowering developers to build robust and adaptable user interfaces.

Aliases point to global values with regard to their meaning or usage and begin with the string cds-alias.

Benefits

  • Simplified Development: Design token aliases provide a clear and consistent way to reference global values, streamlining the development process and reducing the need for manual adjustments.
  • Theme Adaptability: With alias tokens, developers can adapt components to different themes by leveraging predefined mappings to global values. This ensures integration with various design schemes without extensive code modifications.
  • Enhanced Maintenance: By utilizing alias tokens, developers maintain a single source of truth for component attributes, making it easier to update and manage styles across the application. This promotes code reusability and scalability, leading to more efficient maintenance workflows.

Aliases allow a designer, developer, and their application to do things like use a hover state universally without needing to refer to or use a hard coded value. In this case we might look at something like a standard action button. Its default active state is: --cds-alias-status-info and the hover state is: --cds-alias-status-info-shade.

This approach is repeated throughout the Clarity system wherever we need a consistent and repeatable way to specify the states and/or details of a component.

An important feature of the alias token system is that it allows for themes to remap the variables to different global values. For instance, the --cds-alias-status-info token is mapped to var(--cds-global-color-blue-700) in the light theme and var(--cds-global-color-blue-400) in the dark theme. This allows for developers to build components that leverage the alias system to be dark theme compatible with little additional effort.

Component Tokens

In addition to the global and alias tokens explained above, Clarity also contains component-level tokens that assist in mapping specific component attributes to alias tokens. These tokens enable a straightforward way to make one-off changes to components as needed. These tokens begin with the string clr-.

Benefits

  • Fine-grained Customization: Component-level tokens provide precise styling and functionality adjustments.
  • Improved Reusability: They enhance reusability by facilitating easy replication of customized components.
  • Streamlined Maintenance: These tokens streamline maintenance, enabling quick updates for enhanced code maintainability.
