AppFx Translate provides translations for AppFx components. It aims to provide independence and applicability of AppFx components across different products using different localization libraries, or to remain independent of the application's chosen localization library.

Design

AppfxTranslateService

Consumers of the AppFx library should only use the AppfxTranslateService to sync the locale with their application locale.

Features

  • Single locale management point for AppFx components.
  • Simple translate functionality with interpolation arguments similar to @ngx-translate/core.
  • Per-component translations isolation through InjectionTokens.

Limitations & Best Practices

The service does not support:

  • Loading translations from an endpoint.

It is discouraged to:

  • Use the service as a general purpose translate service outside AppFx. It is meant to be a lightweight implementation for AppFx components and lacks general purpose functionalities.
  • Provide global translations. Each AppFx component that needs translations should use its own component providers to achieve isolation.

TranslatePipe

Exposes the translate functionality of the AppfxTranslateService as a pipe.

DateTimeService

Standardized date-time formatting for AppFx components.

  • Limitations: Custom date formats are not supported. To keep the formatting standardized there are only a few different formatting options.

DateTimePipe

Exposes the date-time formatting functionality of the DateTimeService as a pipe.
