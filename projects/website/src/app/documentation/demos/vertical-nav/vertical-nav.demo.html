Vertical Nav is a vertically aligned navigational component. Vertical navigation is a familiar navigation pattern for users. It can fit as many navigation links as needed, scrolling when the content exceeds the viewport.

Usage

Use the vertical navigation when you need:

  • To collapse and expand the navigation to free up space for the content area
  • A hierarchical navigation or nested navigation
  • Many links in the navigation across a top level of navigation
  • Icons to describe navigation items which are helpful when the navigation is collapsed

Layout

Vertical navigation has layout options including dividers and section headers. Active links are displayed with a white background. Touch targets are larger for easier interaction and span the entire width of the navigation, similar to the space of the active link indicator.

Basic

No dividers or section headers.

Dividers

Dividers separate logical clusters of navigation items. Best used when you have two or more links in the clusters.

Dividers and Headers

Dividers and headers are paired to separate logical clusters of navigation items. Best used when you have two or more links in the clusters.

Icons

If you add icons, place them to the left of the top navigation labels. Active items will display a blue icon. Make sure your icons are distinctive and easily recognizable. Touch targets include the entire width of the navigation including the icon.

Hierarchy

Hierarchy is used to show parent-child relationship between navigation items. If a child item is active and the parent item is collapsed, the parent will display as active. When expanded, the active indicator will display on the child. Clicking label text, carets, or icons expands and collapses the parent item.

Basic

Top-level hierarchy items are in a semi bold font weight and have a caret to the right. Child links are normal font weight.

Icons

Icons are applied only to top-level navigation items. Do not add them to child links.

Mixed

Top-level navigation items without child item are semi bold. Clicking navigates a user to a page.

If you need a summary or overview-type landing page for a link group, we recommend you make it as your first child link.

Collapsing the Navigation Bar

The navigation bar itself can be collapsed to allow more space in the content areas. The double caret in the upper-right corner will collapse and expand the navigation bar.

No Icons

When no icons are present, collapsing the navigation will show a basic bar. The entire bar is a click target. Once clicked, the navigation bar expands again.

Icons

  • When collapsed, text will disappear and only icons or icons with carets are displayed. Top-level navigation items with child items show a caret next to the icon. Top-level items without children show no caret next to the icon.
  • Clicking an icon with a caret expands the navigation to the top-level item. Clicking an icon with no caret navigates the user to the corresponding page.
  • An active link is shown as an active icon when collapsed.

Long Labels

When labels are too long, they are trimmed and followed by an ellipsis (…). We recommend that navigation labels remain short and concise to prevent an ellipsis from showing.

Responsive Behavior on Small Screens

When screens are reduced to or drop below 768px wide, the navigation is completely hidden. It can be shown by clicking one of the header icons. All normal vertical navigation designs and behaviors are the same in the responsive state. Read the Responsive section in the Navigation for more information about how this works.

Accessibility

Applications need to inform users who rely on assistive technologies when a routing change loads new content. For this reason, we offer clrFocusOnViewInit along with the Vertical Nav component. The directive allows you to choose which element that assistive technologies should read first when the new content gets loaded. We introduced clrFocusOnViewInit in Clarity v2.1.2, so make sure you are on that or later versions to use the directive.

To let the assistive technologies better track the state of the vertical navigation we recommend adding aria-current="page" to the active link. You will need to add it or remove it depending on which link is currently active. A good marker for that will be the element that has the class .active. The clrAriaCurrentLink directive can be added to any link that uses the routerLink directive to accomplish this.
If the vertical navigation is lazy loaded, the application should do either of the following actions:

  • Move focus to the first navigation link after the vertical navigation is loaded if the user is in the vertical navigation already.
  • Alert assistive technology users in another way (such as with an aria-live announcement).
