@clr/addons/wizard provides a multi-step wizard built on top of Clarity Modal. It adds workflow orchestration, automatic data injection between steps, loading indicators, standardised error handling, conditional step skipping, and an optional "Ready to Complete" summary page.

Wizard Structure

AppFx Wizard is built using Clarity Wizard and provides a lot of additional features. A wizard built using AppFx Wizard has the following structural elements:

  1. Wizard component — hosts <appfx-wizard>, configures steps, defines data retrieval per page, and handles Finish via CloseHandler.
  2. Wizard model — a POJO with Var<T> properties for initial state, page inputs/outputs, and data retrieval.
  3. Page component (per step) — implements StepModelHolder to receive the typed model, and optionally OnStepActivate / OnStepValidate.
  4. Page model (per step) — typed POJO with @In() and @Out() decorated properties.

Features

  • Pre-loads data for the first page; spinner displayed automatically
  • Orchestrates data retrieval on page transitions (lazy or eager)
  • Caches data between back/forth navigation
  • Reloads data when upstream selection changes
  • Optional summary page with step.summary
  • Custom footer component support
  • Conditional step relevance via step.isRelevant
  • Inline (Stepper) mode — same steps, no modal

WizardComponent Inputs

Input Type Description
title string Wizard header title
pages Step[] Ordered step descriptors
wizardModel WorkflowModel Shared data model
opened boolean Two-way open/close binding
size string md / lg / xl / full-screen
closeHandler CloseHandler onSubmit and onCancel callbacks (return Observable)
footer WizardFooterConfig Optional custom footer component class

Step Interface

Property Type Description
title string Step header title
description string? Subtitle shown below the title
navTitle string? Short title in left nav panel
componentClass Type<any> Angular component to render inside the step
model StepModel Typed step data model with @In/@Out properties
mappings Mappings? Custom property name remapping between step and workflow model
isRelevant Var<boolean>? Reactive relevance; step is skipped when false
summary (builder, model?) => SectionModel Builds this step's contribution to the summary page
recreateComponent boolean? Re-instantiate the component on every activation

Step Component Lifecycle Interfaces

Interface Method Description
StepModelHolder model: T Receives the typed step model before activation
OnStepActivate activate(changes) Called every time the step becomes active
OnStepValidate validate(): Observable<boolean> Called when user clicks Next; returning false keeps the step open
